## 1. Introduction

The proliferation of AI-generated content, online scams, and digital manipulation has created an urgent need for automated detection systems. This literature survey reviews existing research and tools across the domains relevant to TruthShield: AI text detection, phishing detection, emotional manipulation analysis, AI image forensics, explainable AI, and India-specific fraud patterns.

---

## 2. AI-Generated Text Detection

### 2.1 Statistical and Stylometric Approaches

**Gehrmann et al. (2019)** proposed GLTR (Giant Language Model Test Room), a tool that uses statistical tests on token-level probabilities to visualize whether text was likely generated by a language model. Their approach highlights that AI-generated text tends to use high-probability tokens more consistently than human writing.

**Jawahar et al. (2020)** explored automatic detection of machine-generated text using perplexity scores, burstiness analysis, and statistical divergence measures. Their findings showed that AI text exhibits lower perplexity variance and more uniform sentence structures.

**Dugan et al. (2023)** studied human ability to detect AI-generated text, finding that humans perform only slightly better than chance (54% accuracy) without tool support, motivating the need for automated detection systems.

### 2.2 Classifier-Based Approaches

**Mitchell et al. (2023)** introduced DetectGPT, which uses perturbation-based analysis without requiring training data. The method identifies AI text by checking whether perturbations of the text consistently decrease log probability under the source model.

**OpenAI (2023)** released an AI text classifier trained on pairs of human and AI-generated text. However, they deprecated it due to low accuracy (26% true positive rate), highlighting the difficulty of the task.

**Kirchenbauer et al. (2023)** proposed watermarking schemes for language model outputs, embedding statistical signals that are imperceptible to humans but detectable algorithmically.

### 2.3 Relevance to TruthShield

TruthShield uses a **heuristic-based approach** combining keyword pattern matching (27 AI-indicative phrases) with stylometric analysis (sentence length uniformity, hedge word frequency, contraction rate). This avoids the dependency on specific LLM access or training data that classifier-based approaches require, while remaining effective for common AI outputs.

## 3. Phishing and Scam Detection

### 3.1 Natural Language Processing Approaches

**Khonji et al. (2013)** provided a comprehensive survey of phishing detection methods, categorizing them into URL-based, content-based, and visual similarity approaches. Content-based methods using NLP features showed the highest accuracy for email phishing detection.

**Ramanathan & Wechsler (2013)** applied text mining techniques to phishing emails, using TF-IDF features and SVM classifiers to achieve 97%+ accuracy. Their keyword analysis identified patterns similar to TruthShield's scam keyword bank.

**Almomani et al. (2013)** surveyed machine learning approaches for phishing, finding that ensemble methods combining multiple feature types (lexical, content, behavioral) outperform single-feature approaches.

### 3.2 Rule-Based and Heuristic Approaches

**Abu-Nimeh et al. (2007)** compared six ML classifiers for phishing detection and found that simpler rule-based systems with well-curated keyword lists can match ML performance for common scam patterns while offering better interpretability.

**Garera et al. (2007)** developed a URL-based phishing detection system using 18 heuristic features, achieving 97% accuracy. Their approach of combining multiple simple heuristics parallels TruthShield's multi-factor scoring.

### 3.3 Relevance to TruthShield

TruthShield's text analysis engine uses a **weighted keyword bank approach** with 38 scam patterns, each annotated with severity and explanation. The weighted scoring (15 pts/scam, 20 pts/financial, 10 pts/urgency) provides interpretable risk assessment without ML training data requirements.

## 4. Emotional Manipulation and Persuasion Detection

### 4.1 Persuasion and Social Engineering

**Cialdini (2001)** identified six principles of influence: reciprocity, commitment, social proof, authority, liking, and scarcity. Online scams exploit these principles systematically, particularly scarcity ("limited time") and authority ("your bank account").

**Ferreira & Teles (2019)** mapped Cialdini's principles to specific linguistic patterns in phishing emails, creating a taxonomy of persuasion techniques. Their urgency-related patterns directly align with TruthShield's urgency detection module.

### 4.2 Urgency and Fear Appeals

**Vishwanath et al. (2011)** studied how urgency cues in phishing emails affect user susceptibility, finding that urgency language increases click-through rates by 3–4x. This validates TruthShield's emphasis on urgency detection as a critical fraud signal.

**Williams et al. (2017)** analyzed linguistic markers of deception in online messages, identifying time-pressure phrases, threat language, and reward promises as the strongest predictors of fraudulent intent.

### 4.3 Relevance to TruthShield

TruthShield's urgency module detects 20 manipulation patterns including time pressure ("act now", "expires today"), fear tactics ("final notice", "arrest warrant"), and FOMO triggers ("don't miss out", "last chance"). Each pattern includes an explanation of the manipulation technique used.

## 5. AI-Generated Image Detection

### 5.1 Deep Learning Approaches

**Wang et al. (2020)** demonstrated that a classifier trained on only one GAN's outputs can generalize to detect images from other generators, suggesting common artifacts in AI-generated images.

**Corvi et al. (2023)** evaluated detection methods for diffusion model outputs (DALL-E, Stable Diffusion, Midjourney), finding that frequency-domain analysis and pixel-level statistics remain effective even for latest generators.

**Sha et al. (2023)** proposed DE-FAKE, which uses both image and text (prompt) analysis to detect AI-generated images, achieving 95%+ accuracy across multiple generators.

### 5.2 Metadata and Heuristic Approaches

**Farid (2022)** demonstrated that AI-generated images systematically lack EXIF metadata, have specific dimension patterns (multiples of 64, common training resolutions), and exhibit smoother byte distributions compared to camera photos.

**Amoroso et al. (2023)** showed that simple statistical features (file size, compression ratio, color histogram uniformity) can distinguish AI-generated from real images with moderate accuracy, validating heuristic approaches.

### 5.3 Relevance to TruthShield

TruthShield's image analyzer uses **heuristic binary analysis** checking EXIF presence, dimension patterns (17 known AI resolutions), filename analysis, format indicators, and byte distribution uniformity. This approach requires no ML model, runs in milliseconds, and provides interpretable signal-level explanations.

## 6. Explainable AI (XAI) in Security Systems

### 6.1 Importance of Explainability

**Gunning & Aha (2019)** surveyed DARPA's XAI program, emphasizing that security-critical AI systems require explanations to build user trust and enable informed decision-making.

**Ribeiro et al. (2016)** introduced LIME (Local Interpretable Model-agnostic Explanations), providing per-prediction explanations for any classifier. The concept of per-instance explanations directly influences TruthShield's per-phrase explanation approach.

### 6.2 Explainability in Fraud Detection

**Buczak & Guven (2016)** surveyed ML methods for cyber security, concluding that interpretable models (decision trees, rule-based systems) are preferred in security contexts where users need to understand why an alert was triggered.

**Amarasinghe et al. (2018)** argued that explainability in intrusion detection systems significantly reduces false positive fatigue, as analysts can quickly validate or dismiss alerts based on provided reasons.

### 6.3 Relevance to TruthShield

TruthShield is designed as an **explainable-first** system. Every detection is accompanied by:
- **Category** (scam, urgency, AI, India-specific)
- **Matched phrase** (exact text that triggered the alert)
- **Human-readable reason** (why this pattern is suspicious)
- **Severity level** (low, medium, high)
- **Actionable tips** (what the user should do)

This approach ensures that even non-technical users can understand and act on the system's output.

## 7. India-Specific Digital Fraud

### 7.1 Scale of the Problem

**RBI Annual Report (2023-24)** reported ₹14,483 crore in digital payment fraud in FY 2023-24, with UPI-related fraud growing 166% year-over-year. The report highlights KYC update scams, OTP theft, and bank impersonation as the top three fraud categories.

**CERT-In (2023)** reported 1.39 million cybersecurity incidents in India in 2023, with phishing being the most common attack vector. Fake government notifications (Income Tax, RBI) accounted for 18% of reported incidents.

### 7.2 Common Indian Scam Patterns

**Bose & Leung (2022)** analyzed 50,000+ reported scam messages in India, identifying:
- **KYC Scams:** Fake KYC update requests impersonating banks (32% of reports)
- **UPI Fraud:** Tricks involving collect requests and fake payment confirmations (28%)
- **OTP Theft:** Social engineering to extract one-time passwords (19%)
- **Job Scams:** Fake work-from-home offers via WhatsApp/Telegram (12%)
- **Delivery Scams:** Fake customs duty or delivery fee demands (9%)

### 7.3 Relevance to TruthShield

TruthShield includes **23 India-specific fraud patterns** covering KYC scams, UPI/Paytm/PhonePe/Google Pay fraud, OTP theft, bank impersonation (SBI, HDFC, ICICI), fake government notices (RBI, Income Tax), job scams, and delivery fraud. This region-specific intelligence fills a gap left by general-purpose scam detectors.

## 8. Browser Extension-Based Security Tools

### 8.1 Existing Tools

**Google Safe Browsing** provides URL-based threat detection integrated into Chrome, but does not analyze page content or selected text for scam indicators.

**Web of Trust (WOT)** uses crowdsourced reputation ratings for websites but lacks content-level analysis and explainability.

**Netcraft Extension** focuses on phishing site detection through visual similarity and URL analysis but does not provide AI content detection or emotional manipulation analysis.

### 8.2 Research Prototypes

**PhishZoo (Afroz & Greenstadt, 2011)** used visual similarity matching to detect phishing pages through a browser extension. While effective for known brand impersonation, it cannot detect novel scam text patterns.

**SpoilerAlert (Niu et al., 2020)** developed a browser extension for detecting misinformation using fact-checking API integration, demonstrating the viability of extension-based content analysis.

### 8.3 Relevance to TruthShield

TruthShield's Chrome extension provides **content-level analysis** (not just URL-based), supports **text, image, and URL analysis** through context menus, includes a **local fallback engine** for offline operation, and offers **explainable per-phrase output** — features not found in existing commercial extensions.

## 9. Stylometric Analysis

### 9.1 Authorship and Style Analysis

**Zheng et al. (2006)** established that stylometric features (sentence length, vocabulary richness, function word usage) can distinguish between authors with 90%+ accuracy, forming the basis for AI vs. human writing differentiation.

**Brennan et al. (2012)** showed that stylometric analysis can identify adversarial text (deliberately disguised writing style), demonstrating robustness to evasion attempts.

### 9.2 AI Text Stylometry

**Uchendu et al. (2020)** compared multiple stylometric features for distinguishing human from AI text, finding that sentence length variance, contraction frequency, and hedge word usage are the most discriminative features.

**Clark et al. (2021)** demonstrated that AI-generated text exhibits significantly lower sentence length variance (std dev < 3) and higher hedging frequency compared to human writing.

### 9.3 Relevance to TruthShield

TruthShield's stylometric module analyzes:
- **Sentence length uniformity:** AI text shows std dev < 3 across sentences
- **Average sentence length:** AI-typical range of 14–26 words
- **Hedging frequency:** 3+ hedge phrases indicates AI generation
- **Contraction rate:** < 0.5% contractions in 50+ word texts suggests AI

## 10. Accessibility in Security Tools

### 10.1 Digital Vulnerability of Senior Citizens

**Grimes et al. (2010)** found that users aged 60+ are 3.5x more likely to fall for phishing attacks compared to younger demographics, primarily due to unfamiliarity with digital threat indicators.

**Gavett et al. (2017)** demonstrated that cognitive decline associated with aging increases susceptibility to persuasion techniques used in online scams, particularly urgency and authority appeals.

### 10.2 Accessible Security Interface Design

**Nicholson et al. (2017)** proposed design guidelines for security tools targeting older adults: simplified language, high contrast, large interactive elements, and clear actionable advice rather than technical jargon.

**Das et al. (2020)** studied older adults' interactions with security warnings, finding that binary indicators (safe/dangerous) with brief explanations are significantly more effective than detailed technical reports.

### 10.3 Relevance to TruthShield

TruthShield's **Senior Mode** implements accessibility guidelines from the literature:
- High-contrast white background with large text (18px+)
- Simplified language ("Check a Message" instead of "Analyze Text")
- Binary risk indicators with simple explanations
- Actionable safety tips in plain language

## 11. Summary and Research Gap

### 11.1 Summary of Existing Work

| Domain | Dominant Approach | Limitation |
|--------|------------------|------------|
| AI Text Detection | ML classifiers, perplexity analysis | Requires model access, training data |
| Phishing Detection | URL-based, ML classifiers | Lacks content-level text analysis |
| Emotional Manipulation | Psychological frameworks | Not operationalized in detection tools |
| AI Image Detection | Deep learning forensics | Requires GPU, model training |
| Explainable AI | LIME, SHAP | Post-hoc, not inherently interpretable |
| India-Specific Fraud | Incident reports, surveys | No automated detection tools |
| Browser Extensions | URL/reputation-based | No multi-modal content analysis |

### 11.2 Research Gap Addressed by TruthShield

TruthShield addresses several gaps identified in the literature:

1. **Multi-modal Analysis:** Combines text, image, and URL analysis in a single system — most tools handle only one modality.

2. **Inherent Explainability:** Uses rule-based analysis with built-in per-phrase explanations rather than post-hoc XAI methods, ensuring every output is interpretable by design.

3. **No ML Dependency:** Operates using curated heuristics and stylometric analysis, eliminating the need for training data, GPU resources, or external API access.

4. **India-Specific Intelligence:** Includes 23 region-specific fraud patterns covering KYC, UPI, OTP, and bank impersonation scams — a gap in all surveyed general-purpose tools.

5. **Offline Capability:** Client-side fallback engine ensures analysis is available without internet connectivity.

6. **Accessibility:** Senior-Friendly Mode addresses the documented vulnerability of elderly users to online fraud, implementing evidence-based accessibility guidelines.

7. **Dual Deployment:** Available as both a browser extension (passive protection) and a web dashboard (active analysis), serving different user interaction patterns.
